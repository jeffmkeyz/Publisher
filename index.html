Guía & Configuración
Cómo funciona

Centro de planificación de contenido. Tú organizas aquí, el servidor publica solo.

  • Creas posts con plataforma, fecha, hora, caption y ruta del archivo
  • Usas ✨ IA para generar el texto adaptado a cada plataforma
  • Exportas el JSON — el servidor lo lee y publica a la hora exacta
  • El script actualiza el estado a published automáticamente
Flujo de estados
BORRADOR→ LISTO→ PROGRAMADO→ PUBLICADO
  • BORRADOR — En construcción
  • LISTO — Caption, archivo y hora confirmados
  • PROGRAMADO — El servidor lo tiene en cola
  • PUBLICADO — Ya está en la plataforma
Plataformas & Automatización
  • TikTok — Bot Playwright, sesión guardada en servidor
  • Instagram — Bot Playwright, sesión persistente
  • YouTube — API oficial Google, más estable
  • Facebook — API oficial para páginas
Convención de rutas
/home/mkeyz/content/videos/nombre.mp4
/home/mkeyz/content/images/nombre.jpg

Lo que escribas en Ruta del archivo es lo que el script Python buscará en el servidor.

Estructura JSON exportado
"id": 1747526400000, // timestamp único "platform": "tiktok", // tiktok|instagram|youtube|facebook "date": "2026-05-20", // YYYY-MM-DD "time": "19:00", // HH:MM (24h) "title": "Título", // YouTube mainly "caption": "Texto...", // caption completo "hashtags": "#trap", // se añaden al caption "media_path": "/home/...", // ruta en servidor "status": "ready" // script busca "ready"

El script solo procesa posts con status: "ready" o "scheduled".

Servidor (cuando lo tengas)
  • pip install playwright google-api-python-client
  • playwright install chromium
  • Sesiones guardadas en /home/mkeyz/browser-profile/
  • Cron revisa el JSON cada minuto y publica si la hora coincide
Tips
  • Planifica la semana completa los domingos
  • Usa ✨ IA para el primer borrador, luego ajusta a tu voz
  • YouTube es la plataforma más estable — empieza por ahí
  • Mejores horas TikTok: 7am, 12pm, 7pm hora local
  • Datos guardados en localStorage — no se pierden al cerrar
