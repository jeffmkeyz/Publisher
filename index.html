Guía & Referencia
Flujo de trabajo

Centro de planificación de contenido. Tú organizas aquí, el servidor publica solo.

  • Planifica la semana completa cada domingo
  • Usa ✨ IA para generar el contenido por plataforma
  • Elige entre 3 versiones y ajusta a tu voz
  • Exporta el JSON — el servidor lo lee y publica a la hora exacta
  • Anota métricas en los posts publicados para alimentar los Stats
Flujo de estados
BORRADOR→ LISTO→ PROGRAMADO→ PUBLICADO
  • BORRADOR — En construcción, falta algo
  • LISTO — Caption, hashtags y hora confirmados
  • PROGRAMADO — El servidor lo tiene en cola
  • PUBLICADO — En la plataforma. Anota métricas aquí
✨ Generación con IA
  • Escribe el tema del post en una línea — no necesita ser largo
  • Elige el tono antes de generar: Street / Lírico / Negocio / Misterioso
  • La IA genera 3 versiones con ángulos distintos del mismo tono
  • Tap en ✓ Usar esta para cargar la versión al formulario
  • Ajusta siempre el resultado a tu voz antes de guardar

Tonos disponibles:

  • 🔥 Street — Directo, crudo, lenguaje de la calle
  • 🎭 Lírico — Poético, metáforas, más artístico
  • 💼 Negocio — CTA claro, orientado a conversión
  • 🌊 Misterioso — Intriga, no revela todo, genera curiosidad
☰ Plantillas
  • Guarda estructuras de caption que funcionan para reutilizarlas
  • Usa variables: {tema} {artista} {genero} {fecha}
  • Filtra por plataforma — solo ves las compatibles con el post actual
  • Combina plantilla + IA: aplica la plantilla y luego refina con IA
＃ Banco de Hashtags
  • Crea grupos por plataforma y género
  • Tap en + Añadir al post — se suman sin borrar lo que hay
  • Puedes combinar varios grupos en un mismo post
  • Recomendado: grupo general + grupo específico del contenido

Estrategia por plataforma:

  • TikTok — Mix trending grandes (#fyp) + nicho musical. Máx 5-7 tags
  • Instagram — 10-15 de nicho específico, sin mega-genéricos
  • YouTube — 3-5 relevantes al contenido exacto
  • Facebook — 2-3 máximo, casi no impactan el alcance
📊 Stats & Métricas
  • Abre el detalle de un post PUBLICADO para anotar sus métricas
  • Métricas disponibles: Views, Likes, Comentarios, Shares, Guardados, Alcance
  • El engagement % se calcula automáticamente
  • 📊 STATS muestra totales, rendimiento por plataforma, mejores días y top posts
  • Con el tiempo revela qué plataforma, horario y tipo de contenido funciona mejor para ti
📋 Copiar al portapapeles
  • En el detalle de cualquier post, tap en 📋 Copiar
  • Copia caption + hashtags listos para pegar directamente en la plataforma
  • Útil para publicar manualmente sin esperar el servidor
Plataformas & Automatización futura
  • TikTok — Bot Playwright, sesión guardada en servidor
  • Instagram — Bot Playwright, sesión persistente
  • YouTube — API oficial Google, más estable para empezar
  • Facebook — API oficial para páginas
Estructura JSON exportado
"platform": "tiktok", // tiktok|instagram|youtube|facebook "date": "2026-05-20", // YYYY-MM-DD "time": "19:00", // HH:MM (24h) "title": "Título", // YouTube mainly "caption": "Texto...", // caption completo "hashtags": "#trap", // se añaden al caption "media_path": "/home/...", // ruta en servidor Linux "status": "ready", // script busca "ready" "metrics": {views,likes...} // métricas reales

El script solo procesa posts con status: "ready" o "scheduled".

Servidor Linux (cuando lo tengas)
  • pip install playwright google-api-python-client
  • playwright install chromium
  • Sesiones del browser en /home/mkeyz/browser-profile/
  • Cron revisa el JSON cada minuto — publica si hora coincide y status = ready
  • Al publicar actualiza el status a published en el JSON
Tips clave
  • Todos los datos viven en localStorage — solo en este dispositivo
  • Exporta el JSON como backup semanal además de enviarlo al servidor
  • TikTok: mejores horas 7am, 12pm, 7pm hora local
  • Instagram: martes-viernes entre 9am-11am y 6pm-8pm
  • YouTube: no importa tanto la hora, más el título SEO y thumbnail
  • Rota las API keys de Gemini desde ⚙ CONFIG si se agota el free tier
